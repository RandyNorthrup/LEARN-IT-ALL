## Upgrade fallout found and repaired

1. The first agent-browser upgrade install correctly failed because the exact allowed postinstall identity still named 0.31.2. The allowlist was updated to 0.32.0 before installation.
2. A full `npm ls --all` then exposed an invalid hoisted peer: Puppeteer's `@puppeteer/browsers` required `proxy-agent >=8.0.1`, while LHCI had supplied 6.5.0.
3. Adding root `proxy-agent` 8.0.2 gives Puppeteer a valid peer while npm keeps LHCI's compatible 6.5.0 nested.
4. `npm ls --all` now exits successfully, and the targeted tree proves both versions are intentionally resolved.
5. Monaco and its React adapter were unused while the learner workspace still used a coding textarea. Both were removed. The current CodeMirror 6 package family now supplies the one tablet/desktop editor path.
6. Ported CodeMirror legacy modes were removed; current official Go parsing replaced the old Go mode, while shell remains labelled text until a current parser is independently accepted.
7. The TypeScript compatibility alias, dual compiler, and host-side learner diagnostics endpoint were removed. TypeScript 6.0.3 is the sole direct compiler because TypeScript 7.0 has no programmatic API and current Next.js tooling did not accept it as a complete TypeScript installation.
8. The PicoC substitute runtime was removed because it covered only C89/C90 and selected C99 while the planned course requires current C23 and sanitizer evidence. No C course can publish until a current runtime passes the full gate.
9. Direct parse5 8.0.1 now supplies current server-side HTML parser-error evidence; Cheerio retains its own compatible parse5 7.3.0 subtree. Keyword-required prose checks were removed, and review-required writing cannot grant automated mastery.
10. Tailwind CSS and its PostCSS adapter moved together from 4.3.2 to 4.3.3; tests, types, strict lint, and production build remain required after this patch upgrade.

## Required compatibility gate

An upgrade is accepted only after:

1. current primary release, engine, peer, browser, migration, and security evidence is reviewed;
2. install scripts are explicitly allowlisted by exact package identity where required;
3. `npm outdated`, `npm ls --all`, and both production and complete audits are understood;
4. focused regression tests pass;
5. full tests, type check, normal lint, strict lint, and production build pass;
6. runtime assets and standalone packaging are present;
7. changed learner flows pass on supported tablet and desktop browsers, plus accessible phone handoff where relevant;
8. replacement decisions and supported-environment exclusions are recorded without shipping an alternate runtime path;
9. Lighthouse remains unrun until the final release gate.
