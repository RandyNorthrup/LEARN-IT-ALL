## Factual Accuracy Spot Check

*Reviewed 65+ questions across all 11 quizzes. Issues listed by severity.*

### CRITICAL — Factual Errors

#### F1. quiz-09 q43 — "Request timed out" is NOT an ICMP message
- **File:** quiz-09-chapter-9.json, question q43
- **Question:** *"Which of the following ICMP error messages help diagnose network connectivity issues?"*
- **Issue:** "Request timed out" is listed as a correct answer and described as an ICMP message. **It is not an ICMP message** — it is a ping utility output indicating the absence of any reply (no ICMP packet received). Actual ICMP message types are: Destination Unreachable (Type 3), Time Exceeded (Type 11), Redirect (Type 5), Echo Reply (Type 0), etc.
- **Fix:** Either remove "Request timed out" from the correct answers and rephrase the question, or change the question to ask about "diagnostic responses/indicators" rather than "ICMP error messages."

#### F2. quiz-06 q5 — Misleading MU-MIMO explanation
- **File:** quiz-06-chapter-6.json, question q5
- **Question:** *"Which Wi-Fi 6 technology allows multiple devices to transmit simultaneously to an access point?"*
- **Correct Answer:** OFDMA (correct)
- **Issue in Explanation:** States *"MU-MIMO allows simultaneous transmission to multiple devices but not from them."* This was true for **Wi-Fi 5 (802.11ac)** which only had downlink MU-MIMO. **Wi-Fi 6 (802.11ax) introduced uplink MU-MIMO**, allowing transmission FROM multiple devices. The correct answer remains OFDMA (which is the primary mechanism), but the explanation's dismissal of MU-MIMO is factually wrong for Wi-Fi 6.
- **Fix:** Update explanation to: *"While Wi-Fi 6 added uplink MU-MIMO, OFDMA is the primary technology for efficient multi-user uplink access through sub-channel resource unit allocation."*

### WARNING — Factual Concerns

#### F3. final-exam q42 — Amber LED attribution
- **File:** final-exam.json, question q42
- **Question:** *"After configuring a new switch, all ports work except the uplink. The uplink port LED is amber. What should you check first?"*
- **Correct Answer:** Port speed/duplex mismatch
- **Issue:** On many enterprise switches (Cisco, HP), an amber port LED specifically indicates **STP blocking state**, not speed/duplex mismatch. A speed/duplex mismatch often results in a **link down** (no LED) rather than amber. The question has "Spanning Tree blocking" as option C, which could be equally or more valid.
- **Mitigation:** The question is ambiguous for exam purposes. Consider rewording the scenario to be more specific about symptoms, or accept multiple valid answers.

#### F4. quiz-08 q34 — MEF E-Access exclusion
- **File:** quiz-08-chapter-8.json, question q34
- **Question:** *"Which of the following are Metro Ethernet service types defined by the MEF?"*
- **Issue:** E-Access is listed as an incorrect option, but MEF does define the E-Access service type (MEF 33). The explanation acknowledges this: *"E-Access is sometimes referenced."* For the CompTIA N+ exam scope, E-Line/E-LAN/E-Tree are the three to know, so this is defensible but the explanation should be clearer.
- **Fix:** Update explanation to explicitly state: *"E-Access is a MEF-defined service but is not one of the three primary service types tested on CompTIA Network+."*

#### F5. quiz-01 q17 — Oversimplified encryption layer attribution
- **File:** quiz-01-chapter-1.json, question q17
- **Question:** *"Which OSI layer handles data encryption?"*
- **Answer:** Layer 6 - Presentation
- **Issue:** While the OSI model assigns encryption to the Presentation Layer, in practice TLS/SSL operates between Layers 4-6 and IPsec operates at Layer 3. The explanation mentions TLS/SSL as a Presentation Layer example, which is a common oversimplification. This is a standard CompTIA exam answer but worth noting as a teaching concern.

#### F6. quiz-09 q45 — Confusing asymmetric routing question
- **File:** quiz-09-chapter-9.json, question q45
- **Question:** *"Which of the following can cause problems with asymmetric routing?"*
- **Issue:** Option E (*"Multiple equal-cost paths in the network"*) is excluded as correct, but the explanation states it "can CAUSE asymmetric routing." The question asks what causes problems WITH it, not what causes it — so the logic is correct, but the wording is confusing and could trick students who understand the concept.

## Recommendations

### Priority 1 — Fix Immediately

1. **Remove duplicate questions in quiz-01:** Replace q33 (HTTPS port duplicate of q7) and q27 (Presentation Layer duplicate of q17) with new questions covering different Chapter 1 topics (e.g., cable types, network topologies, virtual networking).

2. **Fix quiz-09 q43 factual error:** Change "ICMP error messages" to "diagnostic indicators used with ping/ICMP tools" or remove "Request timed out" from the correct answers.

3. **Fix quiz-06 q5 explanation:** Correct the MU-MIMO claim about Wi-Fi 6 uplink support.

### Priority 2 — Standardize Structure

4. **Standardize question IDs:** Add chapter prefixes to all bare IDs in quiz-01, quiz-06 through quiz-10, and final-exam. Pattern: `ch1-q1`, `ch6-q1`, ..., `final-q1`.

5. **Standardize option formats:** Choose either string arrays or object arrays for multiple-select options across all quizzes. String arrays with numeric indices are used in 9/11 quizzes — recommend converting quiz-10 and final-exam to match.

6. **Add missing `tags` fields:** Add to quiz-02 through quiz-05, quiz-10, and final-exam.

### Priority 3 — Improve Quality

7. **Improve distractors:** Replace obviously wrong options in quiz-07, quiz-08, and quiz-09 multiple-select questions with more plausible alternatives.

8. **Standardize points scale:** Either normalize all quizzes to 2-point questions or document the intentional variation in quiz-10 and final-exam scoring.

9. **Clarify ambiguous questions:** Review final-exam q42 (amber LED) and quiz-08 q34 (E-Access) for accuracy/clarity.

10. **Consider scope warnings:** Add metadata tags like `"scope": "beyond-exam"` for questions covering Kubernetes internals, specific container products, or advanced DMVPN details so students know what's supplemental vs exam-critical.
